# Инструкции по деплою на разные хостинги

## Vercel
✅ Уже настроено через `vercel.json`
- Просто задеплойте проект, все работает автоматически

## Netlify
✅ Уже настроено через `netlify.toml`
- Просто задеплойте проект, все работает автоматически

## Apache (обычный хостинг)
✅ Уже настроено через `public/.htaccess`
- Файл автоматически копируется в `dist/` при сборке
- Убедитесь, что на хостинге включен модуль `mod_rewrite`

## Nginx
1. Скопируйте содержимое `nginx.conf` в конфигурацию вашего сервера
2. Или используйте этот файл при настройке Docker-контейнера

## GitHub Pages / Surge.sh / другие статические хостинги
✅ Используйте файл `public/_redirects`
- Некоторые хостинги (например, Surge.sh) автоматически используют этот файл

## Общие шаги для деплоя:

1. Соберите проект:
   ```bash
   npm run build
   ```

2. Загрузите содержимое папки `dist/` на хостинг

3. Убедитесь, что конфигурационный файл для вашего хостинга присутствует в `dist/`

## Проверка после деплоя:

- ✅ Главная страница: `https://ваш-домен.com/`
- ✅ Политика конфиденциальности: `https://ваш-домен.com/privacy`
- ✅ Пользовательское соглашение: `https://ваш-домен.com/terms`

Если страницы возвращают 404, проверьте:
1. Что конфигурационный файл для вашего хостинга присутствует в `dist/`
2. Что на хостинге включены необходимые модули (для Apache - `mod_rewrite`)
